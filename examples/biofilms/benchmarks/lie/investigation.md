# Investigation: Li et al. (2008) synthetic biofilm deformation vs `one_domain.py`

## Goal
Validate the **one-domain diffuse-interface biofilm model** implemented in `examples/utils/biofilm/one_domain.py` against a published deformation-under-flow scenario from the literature, so the setup and parameter scaling are **paper-ready**.

Target literature source (local LaTeX copy):
- `examples/biofilms/benchmarks/lie/main.tex`
  - Mengfei Li, Karel Matouš, Robert Nerenberg (online 27 Feb 2008)
  - “Predicting biofilm deformation with a viscoelastic phase-field model: Modeling and experimental studies”

## Which numerical example is comparable?
From `main.tex`, the most directly comparable test (with enough parameter information) is:
- **Synthetic biofilm deformation under channel flow** (Results & Discussion → “Comparison of experimental and simulated deformation for synthetic biofilm”).

What aligns well with our framework:
- 2D channel flow with no-slip walls.
- A deforming biofilm “blob” fixed at the base (cantilever-like).
- Time-dependent ramp-up of inflow velocity.

What does **not** align directly:
- Li et al. use a **two-fluid phase-field** model coupled to **Oldroyd‑B viscoelasticity** for the biofilm (extra-stress transport).
- Our `one_domain.py` couples a **poroelastic skeleton** (Biot-like) to a single mixture velocity via **Brinkman drag** and uses an Eulerian reference map (`u`) for large deformation kinematics.

## Implementation added in this repo
Benchmark driver created:
- `examples/biofilms/benchmarks/lie/lie_synthetic_deformation_one_domain.py`

Key design choice for robustness and to isolate deformation physics:
- We transport the indicator via the **Eulerian reference map** (“refmap transport”):
  - `alpha(x,t) = alpha0(x - u(x,t))`
  - `phi(x,t) = 1 - (1-phi_b) alpha(x,t)`
  - This means **alpha/phi/S are not solved** in Newton; they are updated after each accepted step.

Tracking output:
- `timeseries.csv` records the **alpha=0.5 contour x-position** at three depths (Li’s “lines 1–3” concept).
  - Important: x-position is computed with **1D linear interpolation** along the y-line (not a nodal threshold), so sub-element displacements are captured.

## Mapping of parameters (Li → one-domain)
Values listed in Li’s Table 1 (synthetic case):
- Fluid: `rho = 1000 kg/m^3`, `mu = 0.001 Pa·s`
- Inflow: `u_avg = 6e-4 m/s`
- Biofilm rheology (reported): `mu_b = 30494 Pa·s`, `G_b = 69736 Pa`

One-domain mapping used in the driver:
- Channel:
  - `H = 10 mm` (from flow cell height; gives `Re = rho*u_avg*H/mu ≈ 6`)
  - Parabolic inflow profile with `u_max = 1.5 u_avg`
  - Linear ramp to full inflow over `t_ramp = 1 s` (paper: ramp over 1 s)
- Skeleton elasticity:
  - `mu_s = G_b`
  - Near-incompressible `lambda_s` from a chosen `nu` (default `nu=0.49`)
- Skeleton viscosity:
  - Kelvin–Voigt term via `solid_visco_eta = mu_b`
- Flow–skeleton coupling:
  - Brinkman drag via `kappa_inv` (tunable; Li’s model does not have permeability)

## What we observed (important for publishability)
### 1) With the Table-1 values as written, deformation is essentially zero
Using (as written in `main.tex`) `G_b ≈ 7e4 Pa`, `mu_b ≈ 3e4 Pa·s` together with `H=10 mm` and `u_avg=6e-4 m/s` produces:
- very small pressure levels (O(1e-2 Pa) on this domain),
- essentially zero skeleton motion (`u ≈ 0` to machine precision) when using typical extension penalties.

This is **not a numerical failure**; it is consistent with the extremely small viscous stresses generated by such a slow flow in a 10‑mm channel when the solid modulus is O(10⁴–10⁵ Pa).

Action item for a paper-quality comparison:
- Re-check the **actual units/values** in the published PDF (OCR of Table 1 is a common source of ×10³ errors).
- Confirm the **effective hydrodynamic length scale** used for `Re` and for stress levels (the flow cell is 10 mm high, but the local gap around the sample may be smaller).

### 2) `gamma_u` (u-extension) is a dominant numerical parameter for small solids
The one-domain reference-map field `u` is continuous over the whole domain and stabilized in the free-fluid region via an extension penalty.

For a small biofilm region on a coarse mesh, `gamma_u` can suppress `u` almost everywhere due to continuity (the solid is “too small” compared to the pinned fluid region).

In practice, for deformation-only tests:
- you often need **smaller `--gamma-u`** than the generic defaults used in microchannel sloughing demos, **or**
- a more deformation-friendly extension strategy (`--u-extension grad` + small `--gamma-u-pin`), **and**
- sufficient mesh resolution in/around the biofilm.

### 3) Displacement extraction must be sub-element (interpolated)
A nodal “rightmost x where alpha>0.5” metric stays *exactly zero* until the interface crosses a grid node.

The driver therefore uses interpolation on each y-line to estimate the `alpha=0.5` crossing location, so tiny displacements are visible in the CSV.

### 4) Quick numerical sanity runs (current driver state)
All runs below used `--backend cpp` and PETSc SNES with LU/MUMPS (the driver default).

**A. Baseline (Table-1 values as written, slow flow, mm-scale channel)**
- Command (short run, VTK enabled):
  - `--nx 60 --ny 24 --dt 0.5 --t-final 1.0 --vtk-every 1 --out-dir out/_tmp_lie_vtk`
- Observation at `t=1.0 s`:
  - `u` magnitude was ~machine-zero (on the order of 1e‑16 in the exported VTK),
  - `timeseries.csv` displacements were effectively 0.

**B. Reduced u-extension penalty + “softened” rheology (illustrates sensitivity)**
- Command:
  - `--nx 60 --ny 24 --dt 0.5 --t-final 1.0 --vtk-every 0 --out-dir out/_tmp_lie_interp --gamma-u 1e-3 --G-b 69.736 --mu-b 30.494`
- Observation at `t=1.0 s`:
  - `dx_line2 ≈ dx_line3 ≈ 7.42e-11 m` (sub-nanometer on this setup).

**C. Larger obstacle + faster flow (still not “paper-matched”, but deformation is measurable)**
- Command:
  - `--nx 60 --ny 24 --dt 0.2 --t-final 2.0 --vtk-every 0 --out-dir out/_tmp_lie_big --u-avg 6e-3 --bio-x0 5e-3 --bio-w 5e-3 --bio-h 4e-3 --gamma-u 1e-3 --G-b 69.736 --mu-b 30.494`
- Observation at `t≈2.2 s` (this run wrote an extra step before terminating):
  - `dx_line1 ≈ 8.87e-09 m`, `dx_line2 ≈ 2.17e-08 m`, `dx_line3 ≈ 2.65e-08 m` (tens of nm).

## How to run
Use the requested environment style:

```bash
conda run --no-capture-output -n fenicsx python -u examples/biofilms/benchmarks/lie/lie_synthetic_deformation_one_domain.py \
  --backend cpp --nx 60 --ny 24 --dt 0.5 --t-final 5.0 --vtk-every 0 \
  --out-dir out/lie_synth_test
```

To diagnose/visualize fields:

```bash
conda run --no-capture-output -n fenicsx python -u examples/biofilms/benchmarks/lie/lie_synthetic_deformation_one_domain.py \
  --backend cpp --nx 60 --ny 24 --dt 0.5 --t-final 1.0 --vtk-every 1 \
  --out-dir out/lie_synth_vtk
```

Outputs:
- `out/lie_synth_*/timeseries.csv`
- `out/lie_synth_*/vtk/step=XXXX.vtu` (if `--vtk-every > 0`)
